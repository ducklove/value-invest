골격은 잘 잡혀 있다. 최상위 4탭(투자정보·종목분석·포트폴리오·국민연금)의 평평한 구조는 이 제품의 사용 사이클 (시장 훑기 → 종목 파기 → 내 자산 관리)과 일치하고, 모바일 하단 탭바 셸과 뷰별 책임 분리도 수준급이다. 문제는 메뉴의 "모양"이 아니라 (a) 만들어 놓고 숨겨둔 기능의 발견가능성, (b) URL·뒤로가기와 단절된 내비게이션 상태, (c) 4번째 탭(국민연금)의 자리 낭비 세 가지다. 셋 다 구조 재설계 없이 소규모 수정으로 해소된다.

  • P1탭 전환이 URL/히스토리에 반영되지 않아 뒤로가기 = 앱 이탈, 새로고침 = 탭 초기화.
  • P1모바일에서는 사이드바가 통째로 사라져 최근 검색·관심 목록을 볼 방법이 없다.
  • P2국민연금(외부 iframe 1장)이 최상위 탭 1/4을 차지 — "도구" 허브 탭으로 교체하면 위 P1 두 건이 함께 풀린다.

4문제점과 개선 제안

구조 문제 (P1) — 메뉴가 기능을 못 따라간다

P1① 스크리너·인사이트 보드로 가는 문이 앱 어디에도 없다

밸류 스크리너는 최근 full-universe 전환·범위 필터까지 넣은 사실상의 주력 기능인데, 진입 경로가 "/labs 카드 → /screener"뿐이고 정작 /labs로 가는 링크가 앱에 없다. 스크리너·인사이트·실험실 3개 뷰 전부 URL을 외운 사람만 쓸 수 있다. 기능 투자 대비 회수가 0에 수렴하는 상태.

근거: index.html:641-652(labs 카드), screener.js:2("/labs 카드 → /screener 직접 URL" 주석), 전역 grep 결과 /labs 링크는 labs 뷰 내부의 "돌아가기" 2곳뿐.
제안: 5장의 "도구 탭" 개편이 정공법. 최소 수정만 원하면 투자정보의 분석 도구 카드 그리드에 '밸류 스크리너'·'인사이트 보드' 내부 카드 2장 추가(외부 도구 카드와 동일한 _extCard 패턴, 0.5일). 실험 기능 배너는 그대로 유지해 신뢰 수준 고지를 지킨다.

P1② 탭 전환이 브라우저 히스토리와 단절 — 뒤로가기가 앱을 종료시킨다

switchView()가 DOM 표시만 바꾸고 history.pushState를 호출하지 않는다. 포트폴리오 → 종목분석 → 투자정보로 이동한 뒤 뒤로가기를 누르면 이전 탭이 아니라 앱 밖으로 나간다. 모바일(안드로이드 백 제스처)에서 특히 치명적이고, 새로고침하면 보던 탭이 사라진다. 딥링크 해석(PATH_TO_VIEW)은 이미 있으므로 "읽기"만 되고 "쓰기"가 없는 반쪽 상태다.

근거: portfolio-shell.js:234-283 switchView() — history API 호출 없음. 감사 보고서 P2 지적과 동일.
제안: switchView에 history.pushState(null,'', '/'+view) 한 줄 + popstate 리스너에서 PATH_TO_VIEW 재사용(재귀 방지용 fromPopstate 옵션). 종목분석은 ?code=까지 동기화하면 "이 종목 보던 화면"도 복원된다. 예상 규모 ~30줄, 1일.

P1③ 모바일에서 최근 검색·관심 목록이 완전히 사라진다

사이드바가 ≤900px에서 display:none !important로 제거되는데 대체 진입점이 없다. "관심종목 등록" 버튼은 모바일에서도 보이지만 등록한 목록을 볼 곳이 없어 기능이 반쪽이 된다. 재방문 사용자의 핵심 재진입 경로(어제 보던 종목)가 모바일에서만 끊긴다.

근거: styles.css:5179 .sidebar { display:none !important; }, 모바일 대체 UI grep 결과 없음. 관심 등록 버튼은 index.html:182.
제안: 종목분석 뷰의 검색창 포커스 시(입력 전) 드롭다운에 "최근 검색 / 관심 목록" 칩 리스트를 보여주는 방식이 가장 자연스럽다(검색 UX 관례, 신규 화면 불요). 데이터·렌더 함수는 사이드바용이 이미 있으므로 재사용. 1~2일.

P1④ 스크리너 안내 문구가 현재 동작과 다르다 (스테일 카피)

스크리너는 finance-pi full-universe 스냅샷으로 전환됐는데(#12), 화면에는 여전히 "분석 이력이 쌓인 종목만 검색 대상입니다"(배너), "종목을 분석할 때마다 스크리너 대상에 추가됩니다"(커버리지 힌트), labs 카드 설명까지 3곳에 옛 동작 설명이 남아 있다. 사용자가 기능 범위를 실제보다 좁게 오해하게 만드는 카피다.

근거: index.html:680, index.html:650, screener.js:65,71 vs 커밋 2857d5d2 "Switch screener to live finance-pi full-universe snapshot".
제안: 문구 3곳을 "전체 상장 유니버스 대상, 재무 데이터 보유 종목 기준" 계열로 교체. 0.5시간.

배치·구성 문제 (P2)

P2⑤ 국민연금이 최상위 탭일 이유가 약하다

4번째 탭은 외부 정적 대시보드(nps-tracker) iframe 1장이 전부다. 게다가 투자정보의 분석 도구 그리드에 이미 국민연금 요약 카드가 있다(자산배분/보유 상위). 앱에서 가장 비싼 자리(모바일 탭바 1/4)를 단일 외부 임베드가 차지하는 동안, 자체 기능인 스크리너는 입구조차 없는 역전 상태다.

근거: index.html:626-628(iframe 전용 뷰), market-dashboard.js:895-902(중복 요약 카드), portfolio-shell.js:296-299("허브는 iframe 임베드만" 주석).
제안: 5장 참조 — 4번째 탭을 "도구" 허브로 교체하고 국민연금은 그 안의 첫 카드로. 기존 /nps 딥링크는 유지(하위 호환).

P2⑥ 포트폴리오 "심층 분석" — 10개 카드 단일 스크롤, 이정표 없음

NAV·리스크·기간 보고서·리밸런싱·배당·일지·평가금액·그룹 비중·자금 입출금·AI 분석이 한 페이지에 세로로 이어진다. 부제("NAV · 리스크 · 리밸런싱 · 현금흐름")가 목차 역할을 흉내 내지만 클릭되지 않고, 모바일에서 특정 카드(예: 배당 캘린더)까지 도달하려면 상당한 스크롤이 필요하다. 성격이 다른 입력 폼(자금 입출금)이 조회용 차트 사이에 끼어 있는 것도 어색하다.

근거: index.html:452-624 (pfPerformanceTab 전체).
제안: 탭 분할보다 상단 고정(sticky) 앵커 칩 내비(NAV·리스크·보고서·리밸런싱·배당·일지·현금흐름·AI)가 적합 — 카드 간 비교 스크롤은 유지하면서 점프만 추가. 자금 입출금은 맨 아래 또는 접힘 기본으로 이동. 1~2일.

P2⑦ 종목분석 — 핵심 데이터가 개인화 블록에 밀려 내려간다

종목을 선택하면 기업정보 → 메모 입력창 → 위키 Q&A → 투자 일지 폼이 먼저 나오고, 정작 이 제품의 정체성인 밸류에이션 차트는 그 아래에 있다. 매번 쓰는 요소(차트)보다 가끔 쓰는 입력 폼 3종이 첫 화면(above the fold)을 차지하는 우선순위 역전이다. 모바일에서 특히 체감이 크다.

근거: index.html:180-214(개인화·Q&A·일지) → index.html:222-233(밸류에이션·연간 지표) 순서.
제안: 위키 Q&A와 투자 일지를 <details> 접힘(제목 줄만 노출) 또는 밸류에이션 아래로 이동. 메모는 1줄 요약 + 클릭 시 확장. 마크업 이동 위주 0.5~1일.

P2⑧ 알림 설정이 3곳에 흩어져 있고 전체를 보는 곳이 없다

종목 알림(종목분석 🔔), 포트폴리오 조건 알림(포트폴리오 🔔), 경제지표 결과 알림(캘린더 🔔)은 각자 맥락에 잘 붙어 있지만, "내가 지금 무슨 알림을 걸어뒀지?"에 답하는 통합 목록이 없다. 알림이 늘수록 잊힌 규칙·중복 규칙이 쌓이고, 해지하려면 원래 화면을 다시 찾아가야 한다.

근거: index.html:183(종목), index.html:324(포트폴리오), 경제캘린더 구독 체크박스(economic-calendar.js).
제안: 프로필 모달에 "내 알림" 섹션(전 종류 규칙 나열 + 개별 삭제 + 원 화면 바로가기). 백엔드는 이미 규칙이 테이블로 있으므로 조회 API 조합이면 된다. 2~3일.

P2⑨ 포트폴리오 탭바에 성격이 다른 컨트롤 5종 혼재

한 줄에 뷰 전환 탭(보유종목/심층 분석) + 패널 토글(액션) + 표시 모드(간편) + 단위 설정(₩/$)이 나란히 있다. 모두 "누르는 것"처럼 보이지만 하나는 화면 전환, 하나는 패널 표시, 하나는 밀도, 하나는 통화라 학습 비용이 생기고, 모바일에서 이 줄이 비좁다.

근거: index.html:298-307.
제안: 탭 2개는 좌측 유지, 액션·간편·통화는 우측에 시각적으로 분리(구분선 또는 아이콘 버튼 그룹). 스타일 조정 수준 0.5일. (간편 토글은 이미 모바일 전용 노출이라 로직 변경 불요.)

문제제안
P3헤더의 "위키 수집량" 배지(index.html:109)는 운영 지표라 일반 사용자에게 소음이다. admin 콘솔로 이동하거나 프로필 모달 하단으로 강등.

P3국민연금 iframe이 탭 진입마다 nonce로 전체 재로드돼(portfolio-shell.js:291) 스크롤·상태가 매번 초기화된다. 세션 내 최초 1회 로드 + "새로고침" 버튼 제공(신선도는 수동 갱신으로).
P3로딩·빈·에러 상태가 동일한 회색 한 줄(.md-loading 재사용)이라 에러를 로딩으로 오인한다. (감사 P2 재확인) 상태 공통 클래스 3종(스피너/빈/에러+재시도) 도입 — 기존 감사 개선 계획과 병합.
P3전역 종목 검색이 없다 — 검색은 종목분석 뷰 안에만 있어, 다른 탭에서 종목을 찾으려면 탭 이동이 선행돼야 한다. 단기: 현행 유지(탭 이동 1회는 수용 가능). 중기: 헤더 검색 아이콘 → 종목분석 검색 포커스 딥링크.
P3모달 9종의 Esc·포커스 동작 불일치. (감사 P1, 미착수 확인) 공통 openModal/closeModal 유틸 + 포커스 트랩 — 기존 감사 체크리스트 P1과 동일 항목이므로 그쪽 일정으로 처리.

5제안 IA 개편안 — "국민연금 탭" → "도구 허브 탭"
