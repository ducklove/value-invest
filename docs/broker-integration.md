# 증권사 계좌 연동 방안

한국 증권사 계좌의 보유 종목·예수금·거래 내역을 프로그램으로 가져오는 방법을
정리한다. value-invest 허브는 Raspberry Pi 위의 단일 사용자 앱이므로, "개인이
자기 계좌를 자동 import" 가능한 경로만을 다룬다. 기준 시점: 2026-07.

## 결론: 통합 연동(한 번에 여러 증권사)은 CODEF만이 유일한 API 경로

"오픈뱅킹으로 여러 증권사 계좌를 한 번에 연동"하려는 접근은 **데이터 범위에서
불가능**하다. 오픈뱅킹은 증권 계좌의 **현금 잔액(예수금/CMA)만** 반환하고,
보유종목·평가금액·거래내역은 아예 없다(KB증권 FAQ가 직접 확인). 보유종목
대시보드가 목표이므로 오픈뱅킹은 구조적으로 쓸 수 없다.

통합(다중 증권사) + 보유종목을 API로 제공하는 서비스는 **CODEF**가 유일하다.
다만 화면 스크래핑 기반(법적 회색지대)이고, 프로덕션은 사업자등록증이 필요하다.
개인이 무료·합법·보유종목까지 얻으려면 결국 **증권사별 공식 API**(한투 KIS 등)를
각각 연결하는 수밖에 없다.

| 경로 | 통합(다증권사) | 보유종목 | 개인 접근 | 비용 | 법적 |
| --- | :---: | :---: | :---: | --- | --- |
| **CODEF (집계 대행)** | ✅ | ✅ | ⚠️ 데모만 | 데모 무료/프로덕션 종량제 | 회색(스크래핑) |
| **증권사별 공식 API** (한투/키움) | ❌ 건별 | ✅ | ✅ | 무료 | ✅ 합법 |
| **마이데이터 앱 + 엑셀 export** | ✅ | ✅ | ✅(수동) | 무료 | ✅ 합법 |
| **오픈뱅킹(금융결제원)** | ✅ | ❌ 현금만 | ❌ 사업자 전용 | — | — |
| **마이데이터 사업자 API 직접** | ✅ | ✅ | ❌ 자본금 5억 | — | — |

## 왜 오픈뱅킹은 안 되는가 (상세)

오픈뱅킹(금융결제원/KFTC)은 두 가지 이유로 보유종목 연동에 **이중으로 막혀 있다.**

### 1. 데이터: 현금 잔액만 있고 보유종목은 없다

오픈뱅킹의 증권 계좌 조회 응답은 `출금가능잔액`/`계좌잔액` 필드만 가진다 —
은행 계좌 잔액과 구조가 같은 **현금(예수금/CMA) 데이터**다. 종목명·수량·평가금액
필드는 존재하지 않는다.

> **KB증권 오픈뱅킹 FAQ**: "다른 금융기관의 수익증권 계좌는 **잔액조회만 가능**
> 하며 **거래내역조회는 불가능**합니다."
> — https://etcimg.kbsec.com/html/design/20201214143317/mo_faq.html

교보증권도 펀드계좌를 "잔고조회만 가능"으로 명시한다. 즉 보유종목·거래내역을
끌어오는 기능 자체가 프레임워크에 없다.

### 2. 접근: 개인은 이용기관에 등록할 수 없다

오픈뱅킹 API를 직접 호출하려면 **이용기관**(핀테크사업자/참가기관)으로 등록해야
한다. 개인(일반고객)은 이용기관이 될 수 없고, 이용기관의 앱을 통해서만 서비스를
받는다. 등록 절차(이용적합성 심사 → 금융보안원 보안점검 → 이용계약)는 B2B
법인 온보딩이다.

- 금융결제원 통합포털: https://openapi.kftc.or.kr/service/openBanking
- 13개 증권사(한투·키움·NH·미래에셋 등)가 2020-12 참가했으나, 이는 **출금계좌
  및 잔액 조회**가 목적이지 보유종목 API가 아니다.

### 결론

오픈뱅킹은 "임금 이체·현금 잔액 확인" 프레임워크다. 보유종목 대시보드에는
데이터도, 접근성도 맞지 않아 **더 이상 검토할 가치가 없다.**

## 통합 연동의 유일한 API 경로: CODEF

[CODEF(코드에프)](https://developer.codef.io)는 여러 증권사 계좌를 하나의
API로 집계하며, **보유종목(종목명·수량·평가금액·평가손익·현재가)**까지 반환한다.

- **보유종목 조회**: [증권사 주식잔고조회](https://developer.codef.io/products/stock/common/a/balance-inquiry)
- **종합 자산**: [증권사 종합자산](https://developer.codef.io/products/stock/common/a/assets)
- **입출금 내역**: [증권사 입출금내역](https://developer.codef.io/products/stock/common/a/transaction)

### 등급·비용

- **데모**: 무료, 하루 100회 호출, 약 1개월, **사업자등록증 불필요**, 실데이터.
  단일 사용자 대시보드(하루 몇 회 폴링, 계좌 몇 개)면 데모로 검증 충분.
- **프로덕션**: 종량제(호출당 과금), 사업자등록증 + 프로젝트 정보 + CODEF 심사.
  [요금플랜](https://www.codef.io/pricing)

### 법적 회색지대 (중요)

CODEF/쿠콘 등 집계 서비스는 전통적으로 **화면 스크래핑**(로그인 자동화) 기반이다.
2022-01-05부터 **마이데이터사업자는 스크래핑이 금지**되고 마이데이터 API
의무화됐다. 다만 비-마이데이터 목적·비-마이데이터사업자의 스크래핑은 규제
경로가 회색지대로 남아있다(2022년 형평성 논란).

→ **비상업적 단일 사용자 Pi 앱**에는 법적 리스크가 낮지만, "완전 합법" 경로는
아니다. 영구적 도입 시 사업자등록 + 마이데이터 연동으로 넘어가야 깨끗하다.

### 쿠콘(Coocon) — CODEF의 대안

은행·증권 계좌 조회를 포함 약 250개 API. 단, 명세가 공개되지 않고 B2B 영업
모델이라 직접 문의가 필요하다. CODEF와 유사한 스크래핑 기반 추정.

## 마이데이터 — 개념은 맞지만 개인 API는 없다

마이데이터 2.0(2025-06-19 개시)이 "전 금융사 자산을 한 곳에서"라는 **개념적
목표에는 정확히 부합**한다. 증권 계좌(계좌잔고·기본정보·연금계좌)를 표준화된
정보제공 API로 노출한다.

- 금투업권 정보제공 API 규격: https://developers.mydatakorea.org/mdtb/apg/mac/bas/FSAG0402

하지만 이 API는 **정보제공자(증권사) → 마이데이터사업자** 사이에만 흐른다.
개인은 사업자(본인신용정보관리업, 자본금 5억 + FSC 인가)가 되어야 직접 호출할
수 있고, 토스·뱅크샐러드는 **공개 개발자 API를 제공하지 않는다.**

개인이 마이데이터를 쓸 수 있는 유일한 방법은 **마이데이터 앱의 수동 export**다:

> **뱅크샐러드**: 앱 → 가계부 → 설정(톱니) → 파일로 받기 → 이메일(Excel/CSV).
> — https://help.banksalad.com/207

수동·이메일 기반이라 Pi에서 자동화하려면 이메일 수신 스크립트가 필요하고,
실시간이 아니다. 다만 **합법·무료·진정한 통합**이라는 장점은 있다.

## 추천 구현 순서

value-invest 허브에 계좌 연동을 도입할 때의 단계. 멀티계좌(accounts phase 1)
기반 위에서 의미가 있으므로 그 이후를 전제한다.

1. **CSV import 파서(증권사 무관)** — 모든 증권사 MTS/HTS의 CSV 내보내기를
   매핑하는 드라이런 검증 레이어. broker key(한투/키움/…)별 컬럼 매핑
   `services/portfolio/broker_import.py`. 가장 보편적 baseline이고 어떤
   API 의존 없이 동작한다.
2. **CODEF 데모 연동(통합 우선 경로)** — 한 번의 연동으로 여러 증권사 보유종목
   집계. 데모(100회/일)로 Pi 검증 → 만족하면 개인사업자 전환하여 프로덕션.
   `services/portfolio/codef_client.py`. 스크래핑 법적 회색지대 인지 필수.
3. **(CODEF 미지원/보완) 증권사별 공식 API** — 한투 KIS 등. CODEF가 못 가져오는
   계좌나 법적 위험을 피하고 싶을 때. 무료·합법이지만 증권사별 개별 연결.
4. **(선택) 마이데이터 앱 export 자동화** — 뱅크샐러드 이메일 Excel을 Pi에서
   수신·파싱. 합법·무료 통합이지만 수동 트리거/비실시간.

## 보안·운영 메모

- API Key/Secret, CODEF 계정 인증서, 마이데이터 앱 연결 토큰은 모두
  **`.env.<profile>`에만** 두고 저장소에 커밋하지 않는다 (`.gitignore` 반영).
- 접근토큰/인증서는 메모리에만 보관(디스크 캐시 금지). 서비스 재시작 시 재발급.
- **조회 전용만** 사용하고 주문/출금 API는 연결하지 않는다 — 쓰기 권한이 노출되면
  계정 정지/법적 리스크가 급증한다.
- CODEF 스크래핑 경로는 계좌 비밀번호를 CODEF 서버에 전달한다 — 신뢰 제3자
  의존을 명시적으로 인지하고, 비상업적 용도 한정으로 사용할 것.
- 백업(`scripts/backup_cache_db.sh`)에 계좌 데이터가 포함되므로 백업 파일
  보관 정책을 함께 점검한다.
